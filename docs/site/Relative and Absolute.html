Relative and Absolute Positioning

Static Positioning

Default setting for everything but HTML. HTML has relative positioning. Offsets are ignored for static positioning.

Relative Positioning

Element is positioned relative to its position in normal document flow. Offset is also relative to normal flow. Offset top / bottom / left/ and right. The original spot of the element is not lost in relative positioning.

Positioning is relative to top bottom etc. So offset top: 50px would move it down from the top by 50px. You can also use negative values, which would move it up from the top.

The first square (light orange) was moved down and to the right using top: 65px; and left: 65px;. As you can see its normal position has been maintained.

Absolute Positioning

All offsets are relative to the nearest ancestor with positioning set (not to static). Absolute positioning takes the object out of the document flow.

If a container is offset, everything within it is relative to it.

The purple container is set to position absolute and give top and left offsets of 0. This will put the purple box at the very top of the page unless the div container is given position absolute or relative. Absolute positioning for the div elements is a mess, so we choose relative.

We also gave the div container a top and left offset, and we can see that the entire container and its contents move together.
