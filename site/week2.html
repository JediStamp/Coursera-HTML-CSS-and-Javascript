Conflict Resolution in CSS

What happens when there are competing style declarations for the same element.

  • Origin Precedence
  • Merge
  • Inheritence
  • Specificity

Origin Precedence

The last declaration wins. (HTML is processed sequentially)/ The stylesheet is in the header. Styles lower than the external link will have precedence over the link.

The external style sheet says that the color for all paragraph elements should be black. But the internal style overrides the declaration for color.

Declarations Merge

If an element has rules for different properties declared in different spots, they merge. IE the background colour can be declared in the style sheet and the text colour in the header, and both will be applied.

In this case the paragraph element in the style sheet also has font-style normal. This is not overridden because it is not declared anywhere else.

Inheritence

Children of an element will also get that property. The beige background specified for the body element is shown for all child elements of the body (except for those which override the background color).

The body element was given a font of Helvetica, meaning all elements have the font Helvetica, except for paragrphs which have the font specified as Cambria below the body font declaration.

Specificity

Most specific selector combination wins. Style element on an element is the most specific. ID is next, then class, then the number of elements used in the selector.

The above paragraph is affected by two combinations in the following order:

  • div.specD p : color bright pink
  • p.specC : color pale pink

Even though the pale pink is specified second, it is less specific, so it is not applied. Both specifications have one class (score 10) but the first has two elements, score 12, while the seond only has 1, score 11. Note: ID would be a score 100, and style in the para itself would be 1000. Number of elements is 1 (for one element). This para is only affected by the first as it does not have a class of "specP".

Important
