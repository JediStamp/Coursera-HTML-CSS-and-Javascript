Box Model & Layout

The outer div has id box, and the inner div has id content. The box has a light blue background and the content has a pale green background.

Padding

Padding is inside the container (and the border). It tells you how close inner elements can get.

Border

The border is 2 px wide.

Margins

Margin is outside the container. It says how close other things can get to the container.

Size

Width was set to 300, but is really, 400 px + 2 * Padding (2 * 20) + 2 * border (2 *2px) + 2 * margin (2 * 40). This is because the size is by default set to the size of the content. (box-sizing: content-box)

We can change this by setting the property box-sizing: border-box;. Now the size to the outside of the margins is 300px.

Box-Sizing is Not Inheritable AKA Star Selector

Some CSS properties cannot be inherited. So putting box-sizing: border-box in the body element does not make it applicable to all elements. We need to use * {} instead of a parent element. This says that every element uses this value.

Cumulative Margins

Left and right margins are cumulative, but top and bottom margins are not. They collapse so that the distance between is the larger of the two margins. So if an element is stacked on top of another, the distance between will be the greater of the two margins. Not the addition of the two.

You can see that the H1 and the box class are 40px apart, not 30px + 40 px = 70 px apart.

To reset browser defaults like padding and margins, you can do this in the start selector as well

Scrolling

If you set the height of the box as well as the width, you can have content spill over. To avoid this, you need to deal with overflow. Auto will put scroll bars in whichever direction they are needed. BUT we all hate double scroll bars. So, don't do it too much... may be acceptable in some situations.
